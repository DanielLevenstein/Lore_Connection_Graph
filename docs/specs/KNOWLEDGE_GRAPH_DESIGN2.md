# Knowledge Graph Redesign

To be useful, we need to redesign the knowledge graph from the ground up.

Because the existing system is so inconsistent, we will not attempt to make the new system backwards compatible with the old.
However, we will support the ability to add a manually created "Character Connections" table at the bottom of character sheets.
When this section is present, it will override whatever data is generated by the character connection graph.

### Specs
- Add entity-specific Attribute graph sections for Session Notes and Places. 
- For extracted attributes ensure that each attribute has an attribute type field.
  - Character and Place attributes should have consistent formatting throughout the whole app.
  - Create a normalized format for names and places
    - If part of the name is unknown, create an id based on the known info like mr_smith, or john_adams_father
  - Suggested fields:
    - id:
    - edge_type: (currently called table)
    - edge_item:
    - edge_value:
    - edge_description:
    - edge_source:
  - Create a maximum length for the attribute description, and if it has to be shortened, make sure shortening is context-aware. 
### Testing
  - Validate attribute value for all characters and place attributes.
    - Don't include and _ characters in name attributes
    - Split the attribute value and description into two fields, ensuring each is of reasonable length. 
  - Scan for duplicate attribute generation.

## Combined Knowledge Graph
Separate the knowledge store of the entity-specific knowledge base from the global knowledge base. 
- Graph nodes for the global knowledge base should only store Character or Place nodes. 
- Graph nodes for the character-specific knowledge should remain unchanged.

### Specifications
- Forbid self-referencing graph nodes.
- Don't show any nodes that don't have any connections to other nodes.
- Remove duplicate edges from the chart
- Only show true people and places in the graph.
- Ensure that unrelated mother and father characters are not combined in the graph.
- Add table to show detailed that are removed from the graph chart. 

### Testing
- Validate that each entity in the graph is either a Person or Place.
- Validate that no duplicate edges are shown in the graph.
- Visually scan the generated graph for entities that don't fit into the above specifications.
